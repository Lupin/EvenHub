# Hermes Agent → G2 Glasses Display App — Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Build an Even Realities G2 companion app that receives Hermes Agent messages (cron outputs, session summaries) and displays them on the G2 glasses. Phase 1 is a passive reader (like Telegram for G2); Phase 2 adds conversation capability.

**Architecture:** Companion app relay — Hermes writes messages to a local JSON file via cron job, the companion app WebView polls it via `fetch()`, renders them as scrollable list items on the G2 display. Navigation: swipe to browse messages, double-tap to exit. Phone companion UI shows message log and status.

**Tech Stack:** TypeScript + @evenrealities/even_hub_sdk 0.0.10 + Vite + even-toolkit CSS tokens. Hermes cron job writes JSON.

### Task 3: Create the data fetcher module

**Objective:** Module that polls the JSON inbox file and returns parsed messages

**Files:**
- Create: `hermes-g2/src/inbox.ts`

**Step 1: Write inbox.ts**

```typescript
import type { HermesMessage, HermesInbox } from './types'

/** Path to the JSON inbox file, served by Vite from public/ */
const INBOX_URL = '/inbox.json'

/** Poll interval in milliseconds */
const POLL_INTERVAL_MS = 30_000  // 30 seconds

/**
 * Fetch the inbox from the companion app's local server.
 * Returns messages newest-first, or empty array on failure.
 */
export async function fetchMessages(): Promise<HermesMessage[]> {
  try {
    const resp = await fetch(INBOX_URL, { cache: 'no-store' })
    if (!resp.ok) return []
    const inbox: HermesInbox = await resp.json()
    return inbox.messages || []
  } catch (err) {
    console.error('fetchMessages failed:', err)
    return []
  }
}

/**
 * Start polling the inbox. Calls onUpdate whenever messages change.
 * Returns a stop function.
 */
export function startPolling(
  onUpdate: (messages: HermesMessage[]) => void
): () => void {
  let lastCount = -1

  const poll = async () => {
    const msgs = await fetchMessages()
    if (msgs.length !== lastCount) {
      lastCount = msgs.length
      onUpdate(msgs)
    }
  }

  // Initial fetch
  poll()

  const interval = setInterval(poll, POLL_INTERVAL_MS)
  return () => clearInterval(interval)
}
```

import {
  CreateStartUpPageContainer,
  RebuildPageContainer,
  ListContainerProperty,
  ListItemContainerProperty,
  TextContainerProperty,
} from '@evenrealities/even_hub_sdk'

const LIST_Y = 20
const LIST_W = 568
const LIST_H = 260
const HEADER_H = 24
const MAX_ITEMS = 20        // SDK limit
const MAX_CHARS_PER_ITEM = 60  // SDK limit

let pageCreated = false

/**
 * Render the header bar (app name + message count).
 */
function renderHeader(bridge: EvenAppBridge, count: number): void {
  const header = new TextContainerProperty({
    containerID: 3,
    containerName: 'header',
    content: `Hermes  (${count})`,
    xPosition: 4,
    yPosition: 0,
    width: 568,

/**
 * Format a message for list display.
 * First line: timestamp + source label
 * Rest: truncated body
 */
function formatMessageForList(msg: HermesMessage): string {
  const date = new Date(msg.ts)
  const time = date.toLocaleTimeString('fr-FR', { hour: '2-digit', minute: '2-digit' })
  const dateStr = date.toLocaleDateString('fr-FR', { day: '2-digit', month: '2-digit' })

  // Source indicator: [CRON], [SES], or [DIR]
  const sourceTag = {
    cron: '[CRON]',
    session: '[SES]',
    direct: '[DIR]',
  }[msg.source]

  // First line of body (title)
  const firstLine = msg.body.split('\n')[0]
  const title = firstLine.length > 40 ? firstLine.substring(0, 38) + '…' : firstLine

  return `${dateStr} ${time} ${sourceTag} ${msg.label}`
}

/**
 * Create a detail text rendering of a single message.
 * Used when user taps a message in the list.
 */
function renderMessageDetail(bridge: EvenAppBridge, msg: HermesMessage): void {
  const date = new Date(msg.ts)
  const dateStr = date.toLocaleString('fr-FR', {
    day: '2-digit', month: '2-digit',
    hour: '2-digit', minute: '2-digit'
  })

  // Truncate body to fit G2 screen (~400 chars)
  let body = msg.body
  if (body.length > 380) {
    body = body.substring(0, 377) + '…'
  }

  const content = `${msg.label}\n${dateStr}\n\n${body}`

  const text = new TextContainerProperty({
    containerID: 1,
    containerName: 'msgDetail',
    content,
    xPosition: 4,
    yPosition: 4,
    width: 568,
    height: 276,
    isEventCapture: 1,  // handles double-tap for back
  })

  bridge.rebuildPageContainer(
    new RebuildPageContainer({
      containerTotalNum: 1,
      textObject: [text],
    })
  )
}

// Module-level cache for messages (so we can show detail on tap)
let cachedMessages: HermesMessage[] = []

/**
 * Render the Level 0 page: message list with header.
 */
export function renderMessageList(
  bridge: EvenAppBridge,
  messages: HermesMessage[]
): void {
  cachedMessages = messages

  const count = messages.length

  // Format messages for list display
  const items = messages.slice(0, MAX_ITEMS).map(formatMessageForList)

  const list = new ListContainerProperty({
    containerID: 1,
    containerName: 'msgList',
    xPosition: LIST_X,
    yPosition: LIST_Y,
    width: LIST_W,
    height: LIST_H,
    isEventCapture: 1,  // handles tap and swipe
    itemContainer: new ListItemContainerProperty({
      itemCount: items.length,
      itemWidth: LIST_W - 8,
      isItemSelectBorderEn: 1,
      itemName: items,
    }),
  })

  if (!pageCreated) {
    bridge.createStartUpPageContainer(
      new CreateStartUpPageContainer({
        containerTotalNum: 1,
        listObject: [list],
      })
    )
    pageCreated = true
  } else {
    bridge.rebuildPageContainer(
      new RebuildPageContainer({
        containerTotalNum: 1,
        listObject: [list],
      })
    )
  }
}

/**
 * Show a single message detail when user taps an item.
 */
export function showMessageDetail(
  bridge: EvenAppBridge,
  index: number
): void {
  const msg = cachedMessages[index]
  if (!msg) return
  renderMessageDetail(bridge, msg)
}

### Task 5: Create the input handler module

**Objective:** Handle G2 touch events — tap to view detail, double-tap to exit

**Files:**
- Create: `hermes-g2/src/input.ts`

**Step 1: Write input.ts**

```typescript
import type { EvenAppBridge } from '@evenrealities/even_hub_sdk'
import { OsEventTypeList } from '@evenrealities/even_hub_sdk'
import { showMessageDetail } from './display'

let currentLevel = 0  // 0 = list, 1 = detail

export function setupInputHandlers(
  bridge: EvenAppBridge
): () => void {
  return bridge.onEvenHubEvent((event) => {
    const evt = event.listEvent || event.textEvent || event.sysEvent
    if (!evt) return

    let type: OsEventTypeList | undefined
    if (evt.eventType !== undefined) {
      type = typeof evt.eventType === 'number'
        ? evt.eventType
        : OsEventTypeList.fromJson(evt.eventType)
    } else if (event.listEvent) {
      type = OsEventTypeList.CLICK_EVENT
    }

    // Level 0 (list): tap item → show detail
    if (currentLevel === 0 && event.listEvent && type === OsEventTypeList.CLICK_EVENT) {
      let idx = event.listEvent.currentSelectItemIndex
      if (idx === undefined || idx === null) idx = 0
      currentLevel = 1
      showMessageDetail(bridge, idx)
      return
    }

    // Level 0: double-tap → exit
    if (currentLevel === 0 && type === OsEventTypeList.DOUBLE_CLICK_EVENT) {
      bridge.shutDownPageContainer(1)
      return
    }

    // Level 1 (detail): double-tap → back to list
    if (currentLevel === 1 && type === OsEventTypeList.DOUBLE_CLICK_EVENT) {
      currentLevel = 0
      // main.ts handles re-rendering the list
      return
    }
  })
}

export function getLevel(): number {
  return currentLevel
}

export function setLevel(level: number): void {
  currentLevel = level
}
```

import { waitForEvenAppBridge } from '@evenrealities/even_hub_sdk'
import { renderMessageList } from './display'
import { setupInputHandlers, getLevel, setLevel } from './input'
import { startPolling, fetchMessages } from './inbox'
import type { HermesMessage } from './types'

let stopPolling: (() => void) | null = null

async function main(): Promise<void> {
  try {
    const bridge = await waitForEvenAppBridge()

    // Show connected status on phone UI
    const g2status = document.getElementById('g2-status')
    const phoneUI = document.getElementById('phone-ui')
    if (g2status) g2status.style.display = 'block'
    if (phoneUI) phoneUI.style.display = 'none'

    // Start polling inbox
    stopPolling = startPolling((messages: HermesMessage[]) => {
      // Update phone count
      const countEl = document.getElementById('msgCount')
      if (countEl) {
        countEl.textContent = `${messages.length} message${messages.length !== 1 ? 's' : ''}`
      }

      // Render on G2 if we're in list view
      if (getLevel() === 0) {
        renderMessageList(bridge, messages)
      }
    })

    // Setup input handlers
    const cleanup = setupInputHandlers(bridge)

    // Listen for back-to-list from detail view
    // We do this by re-polling and re-rendering when level goes back to 0
    setInterval(async () => {
      if (getLevel() === 0) {
        const msgs = await fetchMessages()
        renderMessageList(bridge, msgs)
      }
    }, 5000)  // re-render every 5s as safety net
  } catch (err) {
    console.log('Hermes G2: Bridge not available (running in browser)')
    console.log('This is normal — open on your G2 glasses.')
  }
}

main()

### Task 8: Create the Hermes cron job that writes inbox.json

**Objective:** A cron job that captures Hermes cron outputs and appends them to the G2 inbox

**Files:**
- Create: `hermes-g2/scripts/update-inbox.sh` (shell script)
- Create: `hermes-g2/scripts/update-inbox.py` (Python, for flexibility)

**Step 1: Write the shell script**

```bash
#!/usr/bin/env bash
# update-inbox.sh — Append a message to the Hermes G2 inbox
# Usage: echo "message body" | update-inbox.sh "cron" "job-label"
#
# This script is called by Hermes cron jobs as a post-processing step.
# It reads stdin for the message body, then appends to inbox.json.

set -euo pipefail

SOURCE="${1:-cron}"
LABEL="${2:-Hermes}"
INBOX_FILE="${3:-/Users/gael/Documents/GitHub/EvenHub/hermes-g2/public/inbox.json}"

# Read stdin
BODY=$(cat)

# Use Python for JSON manipulation (reliable across macOS)
python3 -c "
import json, sys, os
from datetime import datetime, timezone

inbox_path = '${INBOX_FILE}'
source = '${SOURCE}'
label = '${LABEL}'
body = '''${BODY}'''

# Load existing inbox or create new
if os.path.exists(inbox_path):
    with open(inbox_path) as f:
        inbox = json.load(f)
else:
    inbox = {'messages': [], 'maxMessages': 50}

# Add new message
inbox['messages'].insert(0, {
    'ts': int(datetime.now(timezone.utc).timestamp() * 1000),
    'source': source,
    'label': label,
    'body': body,
})

# Trim to max
inbox['messages'] = inbox['messages'][:inbox.get('maxMessages', 50)]

# Write atomically
tmp = inbox_path + '.tmp'
with open(tmp, 'w') as f:
    json.dump(inbox, f, indent=2, ensure_ascii=False)
os.replace(tmp, inbox_path)
" || {
  echo "update-inbox.sh: failed to update inbox" >&2
  exit 1
}
```

## Phase 2 — Conversation Mode (Future)

Architecture notes for when the user wants to add voice conversation.

### Overview

Hermes G2 becomes a two-way interface:
1. **Input:** Double-tap (or dedicated gesture) → enters "listen" mode → G2 microphone captures audio → sent to companion app → phone relays to Hermes STT → text sent to Hermes Agent → response → displayed on G2
2. **Output:** Hermes response rendered as text on G2 (single text container, scrollable)

### Technical approach

- **Permission:** Add `"g2-microphone"` to `app.json` permissions
- **Audio capture:** Use `navigator.mediaDevices.getUserMedia({audio: true})` in the companion WebView
- **STT relay:** Send audio blob to Hermes API endpoint (or use browser's built-in SpeechRecognition API)
- **Response display:** `textContainerUpgrade` for real-time streaming, or `rebuildPageContainer` for full response
- **Latency target:** < 5 seconds end-to-end (capture → STT → LLM → display)
- **UX:** Visual indicator (blinking cursor or "listening..." text) while capturing

### Files that would change

- `src/main.ts` — add listen mode state
- `src/display/index.ts` — add listen mode rendering
- `src/input.ts` — add dedicated gesture for voice input
- `src/voice.ts` — new module: audio capture + relay
- `index.html` — companion UI shows recording status
- `app.json` — add `g2-microphone` permission

### Risks

- G2 microphone quality in noisy environments
- WebView audio API availability on iOS/Android
- BLE bandwidth for audio streaming (may need chunking)
- Battery impact of continuous listening mode

## Risk & Tradeoffs

| Risk | Mitigation |
|------|-----------|
| G2 list limit (20 items) | Show newest 20 only; older accessible via companion phone UI |
| JSON file read latency | Use `no-store` cache directive; keep file under 100KB |
| BLE bandwidth for frequent rebuilds | Poll at 30s; use `textContainerUpgrade` where possible |
| Vite dev server must stay running | Document this; for production, serve from static file server or embed in .ehpk |
| Multiple cron jobs writing simultaneously | `os.replace()` in Python is atomic; still, stagger cron schedules |
| Phone WebView cache (pitfall #9) | Kill Vite + Even App between code changes |
